# Git Best Practices for PowerShell
# =====================================
#
# STAGING FILES:
# Use: git add .           (stages all changes in current directory)
# NOT: git add -A          (can hang in PowerShell)
#
# COMMIT MESSAGES:
# Use: git commit -m "Single line message"
# OR:  git commit          (opens editor for multi-line)
# NOT: Multi-line strings in PowerShell (causes hanging)
#
# =====================================
# Commit Message Template
# 
# Format:
# <type>: <subject>
#
# <body>
#
# Types: feat, fix, docs, style, refactor, test, chore
#
# PowerShell Examples (single line):
# git add .
# git commit -m "fix: Resolve layout issues - Fixed aspect ratio - Removed padding"
#
# Or use editor for detailed messages:
# git add .
# git commit  (opens Notepad)
